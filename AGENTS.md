# Agent Guidelines for http-auth

This document provides guidelines for AI agents working on the http-auth project.

## Project Overview

http-auth is a lightweight Python library for HTTP authentication via environment variables.

- Zero dependencies by default (stdlib only)
- Optional dependencies: requests, httpx, keyring
- Python 3.9+
- Uses uv for package management, hatch for multi-version testing, pytest for tests

## Build, Lint, and Test Commands

### Using uv

```bash
# Sync dependencies
uv sync

# Install dev dependencies
uv sync --group dev

# Run all tests
uv run pytest

# Run a single test file (use _test.py suffix)
uv run pytest tests/env_test.py

# Run a specific test function
uv run pytest tests/env_test.py::TestNormalizeHostname::test_lowercase

# Run tests matching a pattern
uv run pytest -k "test_env"

# Run ruff linter (auto-fix where possible)
uv run ruff check --fix src/ tests/

# Run ruff formatter only
uv run ruff format src/ tests/

# Run mypy type checking
uv run mypy src/ --ignore-missing-imports

# Run pyright
uv run pyright src/
```

### Pre-commit

```bash
# Install pre-commit hooks
uv run pre-commit install

# Run all hooks
uv run pre-commit run --all-files
```

### Using hatch (multi-version testing)

```bash
hatch run test
```

## Code Style

### General Python Rules

- Use modern Python 3.9+ syntax
- Use f-strings for string formatting (not `.format()` or `%`)
- Use union operator `|` for types (e.g., `str | None`), not `typing.Union`
- Use PEP-585 generics (e.g., `list[str]`, not `typing.List`)
- Prefer `pathlib` over `os.path`
- Use `argparse` for CLI argument parsing
- Use builtin `itertools` methods where applicable
- Use `encoding="utf-8"` explicitly when opening files in text mode

### Type Annotations

- Use type annotations in function and method signatures
- Do NOT add inline type annotations for local variables when declared and assigned in the same statement
- Use `typing.TYPE_CHECKING` for imports only needed for type hints

### Imports

- Use `from __future__ import annotations` at the top of all files
- Group imports: stdlib → third-party → local
- Sort imports alphabetically within groups
- Use isort with `force-single-line = true`

### Naming Conventions

- Variables/functions: `snake_case`
- Classes: `PascalCase`
- Constants: `SCREAMING_SNAKE_CASE`
- Be creative with dummy data - avoid "Foo", "Bar", and English-only names

### Error Handling

- Use specific exception types
- Never expose secrets in error messages
- Log errors with appropriate levels

### Logging

- Use the `logging` module
- Never use runtime string formatting in log statements
- Use `extra` argument and % placeholders:
  ```python
  logger.info("Token for %s is valid", username, extra={"username": username})
  ```

### Duck Typing

- Prefer duck-typing tests over `isinstance`:
  ```python
  # Good
  if hasattr(obj, 'method'):

  # Avoid
  if isinstance(obj, SpecificClass):
  ```

Test files must use the `_test.py` suffix (pytest convention):
- `tests/env_test.py`
- `tests/resolver_test.py`
- `tests/http_auth_test.py`

## Optional Dependencies

When adding features that require optional dependencies:

```python
from __future__ import annotations

try:
    from requests.auth import AuthBase
except ImportError:
    AuthBase = object  # type: ignore[assignment,misc]
```

## Environment Variable Conventions

- Hostnames are normalized to lowercase
- Periods become underscores (e.g., `example.com` → `example_com`)
- Matching is case-insensitive
- Priority: exact host → suffix (`__`) → global

### Variable Naming

| Priority | Variable | Example Value | Output |
|----------|----------|---------------|--------|
| 1 | `HTTP_AUTH_TOKEN_example_com` | `mytoken` | `Bearer mytoken` |
| 2 | `HTTP_AUTH_HEADER_example_com` | `Bearer token` | `Bearer token` |
| 3 | `HTTP_AUTH_BASIC_example_com` | `user:pass` | `Basic <base64>` |
| 4 | `HTTP_AUTH_TOKEN` | `token` | `Bearer token` |

## CLI Development

The CLI entry point is configured in pyproject.toml:
```toml

http-auth = "http_auth.__main__:main"
```
