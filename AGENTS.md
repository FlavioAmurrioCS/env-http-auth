# Agent Guidelines for http-auth

This document provides guidelines for AI agents working on the http-auth project.

## Project Overview

http-auth is a lightweight Python library for HTTP authentication via environment variables. Key design principles:
- Zero dependencies by default (stdlib only)
- Optional dependencies for requests, httpx, and keyring
- Python 3.9+

## Code Style Guidelines

### General Python Rules

- Use modern Python 3.9+ syntax
- Prefer f-strings for formatting strings (not `.format()` or `%`)
- Use union operator `|` for types (e.g., `str | None`), not `typing.Union`
- Use PEP-585 generics (e.g., `list[str]`, not `typing.List`)
- Prefer `pathlib` over `os.path`
- Use `argparse` for CLI argument parsing
- Use builtin `itertools` methods where applicable

### Type Annotations

- Use type annotations in function and method signatures
- Do NOT add inline type annotations for local variables when declared and assigned in the same statement
- Use `typing.TYPE_CHECKING` for imports only needed for type hints

### Imports

- Use `from __future__ import annotations` at the top of all files
- Group imports: stdlib → third-party → local
- Sort imports alphabetically within groups
- Use isort with `force-single-line = true` (already configured)

### Naming Conventions

- Variables/functions: `snake_case`
- Classes: `PascalCase`
- Constants: `SCREAMING_SNAKE_CASE`
- Be creative with dummy data - avoid "Foo", "Bar", "English-only names"

### Error Handling

- Use specific exception types
- Never expose secrets in error messages
- Log errors with appropriate levels

### Logging

- Use the `logging` module
- Never use runtime string formatting in log statements
- Use `extra` argument and % placeholders:
  ```python
  logger.info("Token for %s is valid", username, extra={"username": username})
  ```

### Duck Typing

- Prefer duck-typing tests over `isinstance`:
  ```python
  # Good
  if hasattr(obj, 'method'):
  
  # Avoid
  if isinstance(obj, SpecificClass):
  ```

## Optional Dependency Pattern

When adding features that require optional dependencies, use this pattern:

```python
from __future__ import annotations
import sys
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    import requests

try:
    from requests.auth import AuthBase
except ImportError:
    AuthBase = object  # type: ignore[misc,assignment]
```

Add extras in pyproject.toml:
```toml
[project.optional-dependencies]
requests = ["requests"]
httpx = ["httpx"]
keyring = ["keyring>=23.0.0"]
all = ["requests", "httpx", "keyring>=23.0.0"]
tests = ["pytest", "http-auth[all]"]
```

## Environment Variable Conventions

When implementing env var parsing:
- Hostnames are normalized to lowercase
- Periods become underscores (e.g., `example.com` → `example_com`)
- Matching is case-insensitive
- Priority: exact host → glob → global

See IMPLEMENTATION_PLAN.md for detailed specification.

## CLI Development
